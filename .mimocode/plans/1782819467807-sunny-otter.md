# Аудит багов auto-click

Проект: Node.js скрипт для автоматизации дашборда `dashboard.tomorrow-school.ai` через Puppeteer + Telegram-бот.

---

## КРИТИЧЕСКИЕ (HIGH)

### 1. `parseInt` NaN —无限循环 (auto-click.js:61-65)
Если `MAX_HOURS=abc`, `parseInt('abc',10)` = `NaN`, а fallback `|| '10'` не срабатывает (строка truthy). `CONFIG.maxHours = NaN` → сравнение `elapsed >= NaN` всегда false → бесконечный цикл. Аналогично `minIntervalMin`/`maxIntervalMin` = `NaN` → `intervalMs = NaN` → `waited < NaN` всегда false → tight loop активности без задержки.

**Фикс**: добавить проверку `isNaN()` после parseInt, или использовать `Number(process.env.X) || default`.

### 2. `startingUp` guard обходится при browser close (auto-click.js:1600-1618)
`state.startingUp = true` ставится на строке 1618, **после** `await state.browser.close()` на строке 1612. Во время await второй вызов `startAutoClick` проходит проверку `if (state.startingUp) return` и оба вызова racing — закрывают/перезапускают браузер одновременно.

**Фикс**: перенести `state.startingUp = true` **до** `browser.close()`.

### 3. Telegram токен в `.claude/settings.local.json` (строка 4)
Токен `8904376054:AAHUbHKFGQ2DO9Aq1DgfKkJTvgd0VkwSkKo` захардкожен в файле, который tracked by git — это credential leak.

**Фикс**: удалить токен из файла, добавить `.claude/settings.local.json` в `.gitignore`.

## СРЕДНИЕ (MEDIUM)

### 4. Callback handlers — fire-and-forget (auto-click.js:747-761)
Все `handleX(...).catch(...)` вызываются без await. Быстрые нажатия кнопок (Start → Stop) вызывают параллельное выполнение — `stopAutoClick` закрывает браузер пока `startAutoClick` ещё в процессе логина.

**Фикс**: либо awaiting, либо добавить mutex/очередь на обработку callback'ов.

### 5. `handleRestart` — detached async IIFE (auto-click.js:367-377)
`state.startTime = Date.now()` сбрасывается до завершения `startAutoClick()`. During login (60+ сек) статус показывает неправильное время. Параллельные команды могут вмешаться.

**Фикс**: await'ить restart логику вместо IIFE, или ставить `startTime` после `startAutoClick()`.

### 6. `activityPromise` не отменяется при stop (auto-click.js:1672-1679)
`stopAutoClick` сразу null'ит промис и закрывает браузер, но `activityLoop` продолжает работу до следующей проверки `state.isRunning`. `performActivity` пытается работать с закрытым браузером → Puppeteer ошибки.

**Фикс**: использовать AbortController или флаг для graceful shutdown.

### 7. Stale element index в DOM query (auto-click.js:1483-1515)
Индексы кнопок берутся в одном `page.evaluate`, а потом используются в другом `page.$$`. Если SPA ре-рендерит кнопки между запросами — кликает не по той кнопке или `undefined`.

**Фикс**: объединить оба действия в один `page.evaluate`.

### 8. `waitForNavigation` на уже загруженном popup (auto-click.js:966)
Если popup загрузился до запуска listener — `waitForNavigation` висит 15 секунд до таймаута.

**Фикс**: проверить состояние загрузки перед `waitForNavigation`, или использовать `Promise.race` с таймаутом.

### 9. HTML injection в Telegram сообщениях (auto-click.js:456, 579)
Текст ошибок интерполируется в HTML-сообщения (`parse_mode: 'HTML'`). Символы `<`, `>`, `&` из ошибок ломают рендеринг.

**Фикс**: экранировать HTML в пользовательском вводе.

## НИЗКИЕ (LOW)

### 10. CDP сессия не закрывается (auto-click.js:906)
`page.createCDPSession()` создаёт DevTools сессию, используется один раз для геолокации, никогда не detatch'ится. Утечка до закрытия страницы.

### 11. Event listeners накапливаются при повторном `setupPage` (auto-click.js:918-933)
При повторном вызове `startAutoClick` на той же странице — `setupPage` добавляет дублирующие `dialog`, `console`, `pageerror` listeners.

### 12. Скриншоты не очищаются (auto-click.js:859-873)
`.screenshots/` директория растёт бесконечно — нет ротации или cleanup.

### 13. `start.sh` удаляет PID файл до проверки (start.sh:21)
PID файл удаляется строкой 21, а строка 28 пытается `cat "$PID_FILE"` — сравнение всегда true → все node процессы убиваются indiscriminately.

### 14. Regex парсинг JSON (auto-click.js:1078)
Redirect URL извлекается regex'ом из raw response вместо `JSON.parse()` — хрупко.

### 15. Credentials видимы в `/proc/<pid>/environ`
EMAIL, PASSWORD, TELEGRAM_TOKEN передаются через env vars — доступны другим пользователям на Linux через procfs. Нет systemd hardening.

### 16. `--no-sandbox` Chromium (auto-click.js:883)
Отключение sandbox'а снижает изоляцию процесса.

### 17. Нет валидации `min <= max` для интервалов
Если `MIN_INTERVAL_MIN > MAX_INTERVAL_MIN` — `getRandomInt` вернёт непредсказуемое значение.

## ПРОВЕРКА ПОСЛЕ ФИКСОВ

1. `node auto-click.js` с `MAX_HOURS=abc` → должен выйти с ошибкой, а не бесконечно крутиться
2. Быстрые нажатия Start/Stop в Telegram → не должно быть race condition
3. `/restart` во время логина → статус должен показывать корректное время
4. `MIN_INTERVAL_MIN=20 MAX_INTERVAL_MIN=5` → должна быть ошибка при старте
